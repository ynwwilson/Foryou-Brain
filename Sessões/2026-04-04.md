# Sessão 2026-04-04 — Migração MegaAPI e Auditoria Concretize IA

## Resumo
Migração completa do webhook da Vercel da Evolution API para a **MegaAPI** (instância SaaS) para o cliente Rodrigo (Concretize Pré Moldados). O sistema agora está funcional, com fluxo completo de mensagens e integração (parcial) com Chatwoot.

## O que foi realizado

### 1. Migração para MegaAPI
- **Substituição da Evolution API:** Removido container e imagem da Evolution API do VPS Hostinger (`76.13.166.51`).
- **Novo Webhook:** Webhook da Vercel (`concretize-ia.vercel.app`) reescrito para ler payloads da MegaAPI.
- **Instância Ativa:** `megastart-Mg0tnMyPBvv` conectada e funcional.
- **Fixes de Endpoint:** Corrigido endpoint de envio para `/rest/sendMessage/${INSTANCE_KEY}/text`.

### 2. Estabilização de Infraestrutura
- **Limpeza de Env Vars:** Identificado erro crítico no Vercel onde variáveis tinham espaços/new-lines invisíveis. Todas as chaves (Supabase, Redis, Anthropic) foram limpas via API.
- **Fila e Lock:** Fila Redis (Upstash) e Distributed Lock (3s batching) funcionando para evitar colisões de mensagens rápidas.
- **Detecção de JID:** Corrigida lógica de identificação do remetente (usando `key.remoteJid` em vez de `body.jid`, que era a própria IA).

### 3. Integração Chatwoot
- **Status:** Implementada, mas atualmente em erro **401 (Unauthorized)**.
- **Problema:** Token `Eus47vLBgDVWagnv2H2RPs8A` é inválido/expirado.
- **Solução:** Rodrigo precisa gerar novo token no perfil de usuário e atualizar no Vercel.
